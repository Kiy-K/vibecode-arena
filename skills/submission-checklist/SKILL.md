# submission-checklist

Before finalizing:

1. Read `/context/challenge.md`.
2. Inspect `/workspace` for the files you changed.
3. Run relevant acceptance checks from the challenge when possible.
4. Fix obvious failures once, then rerun the smallest relevant check.
5. Report remaining risk explicitly instead of claiming success.
